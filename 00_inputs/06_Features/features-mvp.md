# Product Roadmap & Feature Breakdown (P0 / P1 / P2)

## 🚀 P0 Feature Requirements (Core MVP — Immediate Vercel Blueprint)
* **Terminal Interface Layout:** Complete execution canvas utilizing deep black workspace surfaces (`bg-black`), monospace headers, and distinct borders.
* **Autonomous Telemetry Widgets:** Real-time state cards rendering calculated measurements:
  1. Tokens earned/saved in the last hour.
  2. Total repetitive log lines removed.
  3. Total optimization lines sent downstream.
  4. Global output-to-input compression efficiency ratio.
* **Daemon Stream Toggle:** Interface switch element enabling users to instantly pause or restart the simulated log processing stream.
* **Interactive Diagnostic Console:** Terminal-styled output window capable of parsing multi-agent markdown reports with interactive command prompts.

## 🔥 P1 Feature Requirements (Value-Add Interface Elements)
* **Live Stream Activity Monitor:** An animated terminal scrolling pane showing raw log elements cascading into filter frames before metrics update.
* **YAML Source Config Viewer:** Expanding view widget showcasing the mock deployment file configurations (`mockYamlConfig`).
* **Source Cluster Dropdown Selector:** Allowing simulation shifts to toggle telemetry metrics between `k8s`, `cloudwatch`, and `istio`.

## 🌤 P2 Feature Requirements (Extended Horizon Scope)
* **Active Config Multi-Source Management Dashboard:** Add, remove, or modify custom matching regex rule expressions dynamically through an interactive table component form.
* **Export Filter Profiles:** System to export optimized JSON configurations to local developer machines as functional platform setup files.
